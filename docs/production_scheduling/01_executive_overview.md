# 排产系统 · 管理层概览(非技术版)

> 一页纸,面向非技术读者。技术设计见 `00_design_brief.md` 与 `10_process_flow_model_spec.md`。

## 一句话

做一件**连国际主流排程软件都还做不到**的事:计划员只排主干工序,**系统自动补全并排好**所有支持性工作(清洗、配液、装袋)——把排程从"**记录人排好的计划**"升级为"**系统替你排出计划**"。

## 我们的最大不同(核心创新)

生产里大量工作是**支持性**的:设备用前要清洗(CIP)、料要提前配好、反应袋要装好……

- 今天这些**全靠人工一条条挂到工序上**;
- **即便是行业标杆(国际主流的 SuperPro / SchedulePro)也仍然要人工挂**。

我们的做法不一样:你只需要描述**每一步"需要什么、产出什么"**(比如"这步需要一口干净的锅、新鲜的料"),系统就像看懂菜谱一样,**自动推断出"锅脏了要先洗、料过期要重配",把这些支持步骤补全、排进计划**。

> 这是"**记录工具**"和"**决策工具**"的区别——也是一个真正的**技术领先点 / 壁垒**。

## 今天的问题(为什么这件事重要)

人员排班我们已经做得很好;但生产排程几乎**全靠人工**:手填批次开始日期,系统只按模板把时间"盖"出来。设备会不会撞、清洗站够不够、料会不会过期、人手够不够——**系统都不检查**,冲突往往**到现场才暴露**;很多排程还停留在**白板拍照、Excel 和老师傅的脑子**里。

## 顺带带来的能力

- **灵活贴合现实(我们叫"钉子+弹簧")**:钉死几个关键日期,中间工序能伸缩,系统按真实空档安排;同一产品 9 天 / 11 天周期能**自动调整**,而非死板模板。
- **提前防冲突、并给出路**:把冲突**提前**标红、说清原因;实在排不下时提示"**这里要加一台设备 / 一个班次**",而不是给一份做不到的计划。*(尤其清洗站、配液罐这种隐形瓶颈——真实数据里某天挤了 16 次清洗。)*
- **计划随现实走**:某步晚了一小时,录入实际,后面**自动调整**。
- **合规可追溯**:关键步骤**绝不偷偷顺延**,每步"为什么这么排"都能解释——符合 GMP / 审计。

## 价值与风险(实话实说)

- **价值**:省人力、少漏排、提前防冲突、产能与交期更稳、用人更平滑;且是**独立新模块、不动现有系统、复用已有技术**,推进**低风险**。
- **风险**:正因为"自动补全"是别人没做过的,**没有现成产品可抄**,这套自动推断的规则需要我们**自己定义并充分验证**;大规模排程的计算效率需专门处理(已有成熟应对路线)。
