# WBP2486 全程走查(USP→DSP)· 工艺流模型压测 v0.1

> 目的:用真实的 WBP2486 一整条 USP→收获→DSP(含配液)在模型里走一遍,验证表达力 + **主动暴露缺口**(§6 喂给后续缺口审计)。
> 数据来源:USP 模板7「WBP2486/B」、DSP 模板10/27、`WBP2486_DSP_Process_Flow.md`、车间排产板提取(`outputs/downstream_schedule_*`)。模型见 `10_process_flow_model_spec.md`。

## 1. 主链(钉子 + 弹簧 + 阶段)

**USP(模板7,~40 天,不可中断):**
- 🔩 **钉子①** 复苏 @ USP-Day0(OP-00001,6h/3人)
- Wave 接种(7h/2人)→ SUB50 → SUB250 → SUB1000 → SUB2000 种子串(各级间"最短/最长培养时距"= 弹簧,由生物学定)
- 4000L ABEC:培养基灌注(OP-00048,4h/3人)→ 接种(OP-00050,4h/2人)
- 〰️ 培养(弹簧,~10 天)
- 🔩 **钉子②** 收获(OP-00051,3h/2人)@ ~USP-Day32

**接力(= pull + 时序):** 收获产出「收获液」令牌 → DSP 的 AC 消费它 + "收获后 ≤X h 进 AC"时序约束(对应模板27 day-link:USP-Day32 → DSP-Day1)。

**DSP(模板10/27,~17 天):** AC → VIN → CEX → AEX → HA → UFDF1/2 → VF → UFDF3 → Bulk Fill;各步之间的「中间池」令牌带 hold time(= 弹簧上下界)。

## 2. 生成规则在跑(钩子)

| 规则 | 类型 | 在 2486 里 | 模型表达 |
|---|---|---|---|
| 每日取样 | push 跨度 | 绑"接种→收获"弹簧,每日,±8h;次数=培养天数 | ✓ 正中 §3.7 |
| 每日/隔日补料 | push 跨度 | 同类,周期=每 N 天 | ✓ |
| 装袋(SUS 反应器) | pull | 4000L ABEC 需 袋=installed → 派生"反应器前期准备"包(装袋+完整性测试+电极安装) | ✓ |
| CIP/SIP(SS 设备) | pull | DSP 储罐/层析 skid用前需 clean/sterile → 派生 CIP→SIP;用后 dirty 受 DHT 限须窗内清洗 | ✓(DSP Day3-7 清洗成簇即此涌现) |
| 配液 | pull | 灌注需 prepared 培养基(PT-00008);DSP 各步需 buffer(PT-00009)→ 派生配液,产物带效期、可分装 | ✓ |

## 3. 物料令牌 + 效期

培养基(~24h)、各 buffer(24–72h)、收获液(短 hold 到 AC)、DSP 各步中间池(各自 hold)。**全部 = 生产者→消费者的 min+max lag**,超期 = 时序不可行(免费检测)。✓

## 4. 设备状态机(按类不同 → 验证 D6)

- **ABEC 4000L(SUS)**:袋 {无 → installed → used};**无 CIP/SIP**。
- **SS 储罐/层析 skid(T181x、AKTA 1850/1851)**:洁净{dirty↔clean,CIP,DHT/CHT}、灭菌{SIP}。
- **CIP 站、配液罐 BH17xx**:共享有限资源(并发上限)→ 瓶颈,留资源层。

→ 同一"状态机即算子图"机制,三类设备各落不同图。✓

## 5. 一段派生展开(示意:DSP 的 CEX 步要开始)

```
目标:CEX 层析步可执行
 ├ 需 CEX 层析 skid(SS)@ clean ∧ sterile
 │    └ sterile ← SIP(前提 clean) ← clean ← CIP   ⇒ 派生 CIP→SIP
 ├ 需 CEX buffer @ prepared ∧ 在效期内
 │    └ ← 配液(PT-00009)产出 buffer 令牌(效期 X h)  ⇒ 派生配液(+其配液罐又需 clean→派生 CIP)
 └ 需 上游 AEX 池令牌(= AEX 步产出),且"AEX 产出后 ≤Y h 必须进 CEX"
      └ pull AEX 池 + max-lag(效期/零等待)
 ⇒ 自动摊开:CIP(skid)→SIP(skid) ‖ CIP(配液罐)→配液(buffer) ‖ AEX 池(限 Y h 内)  → CEX
```

## 6. ⚠️ 模型吃紧 / 候选缺口(待缺口审计复核)

1. **"循环 N 次"类重复(非日历)**:DSP 的 `AC Cycle 1-4`、层析多循环 —— 这是"**重复 N 次**",既非日历周期、也非需求 pull。v1 跨度规则只支持**日历/周期** → **可能表达不了"循环 N 次"**。需定:循环算 N 个显式操作?还是补一种"计次重复"规则?
2. **QC 取样结果的分支 / 放行门**:取样送 QC,结果可能 gate 下游(放行才继续 / 不合格则 hold/返工)。模型**不表达条件分支/决策**(研究指出会跨入 NP-hard 的 CSTNU)。→ **放行门 / 不合格返工 目前是缺口或需显式简化**。
3. **数量 / 物料平衡**:补料量、配液量、分装份数随批量变。v1 归批次层,但"取样/补料的**量**"工艺流层要不要带?待定。
4. **公用资源 WFI**:CIP 成簇时 WFI 容量上限 —— 已知,归资源层。
5. **种子升罐按"活细胞密度(VCD)"而非纯日历**:种子何时升罐有时看 VCD/倍增,而非固定天数 → 又是"按工艺进度触发",v1 不做 → 缺口或简化为日历。
6. **设备态 × 效期令牌膨胀**:SS 设备 ×{clean/dirty/sterile}× 效期 —— 规模可控但要留意(§11 守则已提)。
7. **不可中断的长培养(USP)与排程**:USP 14–21 天不可中断,是一根超长"刚性弹簧";与下游滚动重排、冻结窗口如何相处,待批次层细化。
