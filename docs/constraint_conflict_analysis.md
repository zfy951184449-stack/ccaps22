# 约束冲突分析：季度±4h vs 月度±8h

## 新约束定义

### 季度约束
- **标准工时**：动态计算（工作日 × 8小时）
- **浮动范围**：±4小时（硬约束）
- **约束范围**：[标准工时 - 4h, 标准工时 + 4h]
- **示例**：标准工时500小时 → 约束范围 [496h, 504h]

### 月度约束
- **标准工时**：动态计算（月度工作日 × 8小时）
- **浮动范围**：±8小时（硬约束）
- **约束范围**：[标准工时 - 8h, 标准工时 + 8h]
- **示例**：标准工时166.64小时 → 约束范围 [158.64h, 174.64h]

## 数学冲突分析

### 场景1：所有月份都达到月度上限

假设：
- 季度标准工时：500小时
- 月度标准工时：166.64小时
- 季度约束：496-504小时
- 月度约束：158.64-174.64小时

如果3个月都达到月度上限：
- 10月：174.64小时
- 11月：174.64小时
- 12月：174.64小时
- **季度总计**：174.64 × 3 = **523.92小时** > **504小时**（季度上限）❌

**结论**：存在冲突！

### 场景2：所有月份都达到月度下限

如果3个月都达到月度下限：
- 10月：158.64小时
- 11月：158.64小时
- 12月：158.64小时
- **季度总计**：158.64 × 3 = **475.92小时** < **496小时**（季度下限）❌

**结论**：存在冲突！

### 场景3：月份工时分布不均

如果月份工时分布不均：
- 10月：174.64小时（月度上限）
- 11月：174.64小时（月度上限）
- 12月：154.72小时（低于月度下限）
- **季度总计**：174.64 + 174.64 + 154.72 = **504小时**（季度上限）✅
- **但12月低于月度下限**：154.72 < 158.64 ❌

**结论**：存在冲突！

## 解决方案

### 策略：季度约束优先

1. **季度约束优先**：确保季度约束必须满足
2. **月度约束在季度约束允许的范围内满足**：如果季度剩余容量不足，调整月度约束
3. **动态调整月度上限**：根据季度剩余容量动态调整月度上限

### 实现机制

#### 1. 月度均衡时检查季度约束

```typescript
// 检查季度约束，确保月度补足不会导致季度超限
const quarterHours = calculateAccumulatedQuarterHours(employeeId);
const quarterRemainingCapacity = QUARTER_MAX_HOURS - quarterHours; // 季度剩余容量

// 如果月度补足会导致季度超限，调整月度上限
const adjustedMonthMaxHours = Math.min(
  monthMaxHours,
  monthHours + quarterRemainingCapacity // 不超过季度剩余容量
);
```

#### 2. 月度补足时限制补足量

```typescript
if (monthHours < monthMinHours) {
  // 需要增加工时到最低要求，但不超过季度剩余容量
  const neededHours = monthMinHours - monthHours;
  const diff = Math.min(neededHours, quarterRemainingCapacity); // 不超过季度剩余容量
  
  // 如果季度剩余容量不足，记录警告
  if (diff < neededHours - 0.1) {
    warnings.push(
      `员工${employeeId}在${monthKey}月需要补足${neededHours.toFixed(2)}h，但季度剩余容量仅${quarterRemainingCapacity.toFixed(2)}h，只能补足${diff.toFixed(2)}h（季度约束优先）`
    );
  }
}
```

## 冲突处理结果

### 情况1：季度剩余容量充足

- **季度工时**：490小时（低于下限496小时）
- **季度剩余容量**：504 - 490 = 14小时
- **10月工时**：158小时（低于下限158.64小时）
- **需要补足**：158.64 - 158 = 0.64小时
- **季度剩余容量**：14小时 > 0.64小时 ✅
- **结果**：可以补足到月度下限

## 结论

### 是否存在冲突？

**数学上存在冲突**，但通过以下机制可以避免：

1. ✅ **季度约束优先**：确保季度约束必须满足
2. ✅ **动态调整月度上限**：根据季度剩余容量动态调整
3. ✅ **限制月度补足量**：不超过季度剩余容量
4. ✅ **警告机制**：当季度剩余容量不足时记录警告

### 实际效果

- **季度约束**：始终满足（硬约束）
- **月度约束**：在季度约束允许的范围内尽量满足
- **冲突处理**：季度约束优先，月度约束可能无法完全满足（会记录警告）

### 建议

如果发现月度约束无法满足的情况较多，可以考虑：
1. 调整季度浮动范围（例如：±6小时）
2. 调整月度浮动范围（例如：±6小时）
3. 优化季度补足策略，按月份均匀分配
