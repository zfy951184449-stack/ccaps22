# 生物制药 CMO 领域地图

本文件提供生物制药 CMO 排程相关的基础术语、关键约束和开发落地红线，供后端、前端和 solver 共同对齐。

## 1. 核心层级与术语

- **Campaign（生产活动）**：针对某一特定药品的连续多批次生产计划。不同 Campaign 之间极度防范交叉污染，切换时通常需要长达数周的 Product Changeover。
- **Batch（批次）**：单次完整的投料到产出循环。同一种产品的不同批次之间通常只需要较短的 Batch Turnover。
- **Stage（阶段）**：
  - **USP（上游）**：细胞复苏、扩增、发酵补料。通常持续 14-21 天，不可逆、不可中断。
  - **DSP（下游）**：离心、层析、超滤/透滤、病毒灭活等。步骤密集，通常集中在几天内完成。
- **Ancillary Tasks（辅助工序）**：
  - Media Prep、Buffer Prep：必须在主节点前完成，并受物理/化学有效期约束。
  - 环境监测、设备校验等独立任务。

## 2. 关键业务约束

### 2.1 时间窗与零等待

- 中间产物可能快速降解。上一步结束后，下一步必须在限定时间内启动，某些场景要求零等待。
- 配制好的缓冲液通常有 24-72 小时的使用时效。
- 在求解器中不能只写 `EndBeforeStart`，还需要明确建模最大 hold time。

### 2.2 公用资源瓶颈

- 约束排产的常常不是主设备，而是 WFI、CIP 等公用系统容量。
- 多车间同时做大流量 CIP 或配液可能抽空公用流体能力。
- 这类约束应被建模为共享资源的容量约束，而不是事后人工顺延。

### 2.3 空间隔离与人员资质

- Suite 级别通常要求互斥，尤其要严格隔离 pre-viral / post-viral 区域。
- 人员除了岗位技能外，还受洁净级别准入和更衣耗时约束。
- 跨区域操作应显式预留 setup / gowning 时间。

### 2.4 设备状态流转

- 设备存在 DHT（脏停放时间）和 CHT（洁净有效期）等倒计时约束。
- 设备状态不是简单的 Idle / Busy，而是带时间衰减的状态机。

## 3. 开发落地红线

1. 生物制药排程默认不能“自动修剪”不合理时间间隔；冲突应返回明确失败或 `Infeasible`。
2. 前后端和 solver 必须统一使用标准工艺术语，如 `cip_duration`、`max_hold_time`、`usp_step`、`dsp_step`、`wfi_demand`。
3. 复杂清洗、换线、倒计时约束应封装成独立模块，不要堆在主流程里。
