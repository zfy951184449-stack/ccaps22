# 工艺模版编辑器 — 备选方案存档

> **状态**：已选定方案 F（资源甘特图），以下为讨论阶段的备选方案存档。
> **日期**：2026-05-06

---

## 方案 A：表格驱动（Notion / Airtable）

**核心理念**：结构化数据精准编辑，无时间轴

| 维度 | 评价 |
|------|------|
| ✅ 优势 | 批量编辑最快、精确数值输入、键盘导航、筛选排序、信息密度最高 |
| ❌ 劣势 | 看不到时间关系、约束关系只能用符号暗示、缺乏空间直觉 |
| 适合 | 大量操作（60+）的快速录入和属性修改 |
| 实现难度 | ⭐⭐ 低 |

---

## 方案 B：节点画布（n8n / Blueprint）

**核心理念**：操作 = 节点，约束 = 连线

| 维度 | 评价 |
|------|------|
| ✅ 优势 | 约束编辑最佳：拖拽连线创建约束、共享组可视化、关键路径高亮 |
| ❌ 劣势 | 不表达时间轴、60 个节点布局拥挤、属性编辑需侧边栏 |
| 适合 | 约束关系密集的模版、理解操作依赖结构 |
| 实现难度 | ⭐⭐⭐ 中（需 React Flow） |

---

## 方案 C：时间线 NLE（Premiere Pro）

**核心理念**：Stage = 轨道，操作 = 可拖拽片段

| 维度 | 评价 |
|------|------|
| ✅ 优势 | 时间关系最直觉、从操作库拖入添加、裁剪 = 调工时、并行操作一眼可见 |
| ❌ 劣势 | 本质是改进版甘特图、约束线在轨道间画会乱 |
| 适合 | 时间密集型模版 |
| 实现难度 | ⭐⭐⭐⭐ 高 |

> ⚠️ 方案 C 和现有甘特图区别主要在交互范式，核心布局仍是 Stage×Time。

---

## 方案 E：混合泳道（自研融合）

**核心理念**：泳道 + 属性 + 仪表盘同屏

| 维度 | 评价 |
|------|------|
| ✅ 优势 | 编辑+预览+反馈同屏、泳道保留时间直觉、仪表盘实时反馈 |
| ❌ 劣势 | 屏幕分割多、移动端不友好、实现最复杂 |
| 适合 | 专业用户的主力工作台 |
| 实现难度 | ⭐⭐⭐⭐ 高 |
