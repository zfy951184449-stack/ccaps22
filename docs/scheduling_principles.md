# 排班原则记录

> 本文仅整理已明确的排班原则/需求，用于指导 v4 智能排班策略的实现与排查。各条内容均来自近期沟通，不作延伸或再阐述。

## 工时约束
- **季度工时**：必须大于或等于季度标准工时，禁止低于目标值。
- **月度工时**：需保持在月度标准工时的上下浮动范围内，默认 ±8 小时，该容差可在前端配置。

## 夜班管理
- 夜班的判定依据班次编码（包含 “NIGHT”）或运行时间段（开始 ≥19:00、跨日至凌晨或持续超过夜班标称工时）。
- 夜班结束后应安排至少 2 天休息；除非特殊紧急情况，不应出现仅休 1 天就再次排班的情形，如遇此类情况需记录并优先调整。

## 补班策略
- 补班不局限于工作日；在满足工时和约束前提下，可在普通周末安排补班以支援产能。
- 对于需支付 3 倍工资的法定节假日，补班需受到专门管控：应识别节假日日期、单独计成本，并只在获得批准时排班。

## 休息日管理
- 在具备条件的情况下，安排休息日应尽量保持连续，避免出现碎片化的休息安排影响恢复质量；夜班后优先保证连续两天休息。
