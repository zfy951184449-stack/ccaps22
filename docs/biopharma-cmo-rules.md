# 生物制药 CMO 排程语义参考

本文件沉淀较深的生物制药 CMO 排程语义，用于数据库设计、API 建模、solver 约束设计和劳动力调度的一致性对齐。

## 1. 生产层级与生命周期

- **Campaign**：同一分子的连续多批次生产。Campaign 之间通常需要长时间换线、VHP、残留检测等。
- **Batch**：单次端到端生产循环，强调谱系追踪和上下级物料溯源。
- **USP**：Seed Train 到 Production Bioreactor，长周期、不可中断、强依赖无菌环境。
- **DSP**：离心、层析、UF/DF、病毒处理、原液灌装，步骤密集，对人力和缓冲液消耗集中。
- **Ancillary**：Media/Buffer Prep 等辅助工序，受储罐、称量间、物料有效期约束。

## 2. 时间窗与流转约束

- **Zero-Wait / Hold Time**：关键中间产物和配液存在严格时效；违反时不应靠静默顺延掩盖。
- **DHT / CHT**：设备使用后和清洗/灭菌后的倒计时会改变后续可行性，不能当作普通 idle 处理。

## 3. 资源与空间瓶颈

- **Utilities Capacity Leveling**：WFI、PW、CIP/SIP 工位等公用系统都可能成为并发瓶颈。
- **Spatial Segregation**：Suite 互斥、pre-viral / post-viral 隔离、HVAC 约束都应在建模层显式表达。
- **Personnel & Gowning**：技能匹配之外，还要建模跨区更衣、消毒、交接班重叠时间。

## 4. 质量控制与物料约束

- 关键步骤可能依赖 QC release，不能默认“上一步完成后下一步立刻可做”。
- 必须区分 SUS 与 SS 设备：SS 受 CIP/SIP、DHT/CHT 约束，SUS 更偏向 setup 与完整性检查成本。

## 5. 建模红线

1. 时间窗冲突应在模型层显式失败或成为清晰惩罚项，不允许静默修正。
2. 设备、人员、区域状态要用细粒度状态机表达，不能简化成过粗的 idle/running。
3. WFI 削峰、DHT/CHT、Suite 互斥等复杂语义应封装成独立约束模块。
4. API 字段、响应结构、变量命名应统一采用生物制药规约词汇。

## 6. 劳动力与班次调度

- 资质不只是岗位技能，还包括洁净区准入、SOP 培训是否有效。
- 跨控制区操作必须插入 gowning / de-gowning 时间。
- 连续工艺需要交接班重叠窗口，避免脱岗盲区。
- 轮班模式要纳入连续工作天数、最小休息时长、连续夜班数等合规约束。
