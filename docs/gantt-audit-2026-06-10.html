一、链A：工艺模版甘特

P0

P0bug已修复A1 · 拖动阶段/设备组条整体平移后「假成功」——提示已移动但从不落库

现象：拖动阶段（或设备分组）的汇总条，组内工序跟着平移，松手弹「已移动 N 个任务 +Xh」撤销提示；但没有任何请求发出，下次刷新全部弹回原位。

根因：wxb-ui/GanttChart/useGanttDrag.ts:258-286 组拖拽结束时 onGroupDragEnd 为 undefined 就跳过持久化，却仍无条件弹成功 toast；ProcessTemplateV3/ProcessTemplateV3Editor.tsx:904-910 只传了 onTaskDragEnd / onTaskResizeEnd，从未传 onGroupDragEnd。

对照：批次甘特 BatchGanttV4/index.tsx:635 是传了 onGroupDragEnd 的，只有模版编辑器漏接。

建议：实现 onGroupDragEnd（按 deltaHours 批量 updateStageOperation 后刷新）；或未接通前禁用组拖拽且不弹成功提示。

已修复（2026-06-10）：useV3EditorActions 新增 handleGroupDragEnd（Promise.allSettled 并发落库 + 单次刷新 + 部分失败提示「N 条中 M 条失败」）并接入 WxbGanttChart；useGanttDrag 防御性修复——无 onGroupDragEnd 时只做画布 cleanup，不入 undo 栈、不弹成功提示；撤销 toast 走反向回调同样持久化，阶段/设备两种视图通用。

P1

P1bug已修复A2 · 新建操作的「时间窗」输入被后端静默覆盖

现象：新增操作弹窗里手动填的时间窗保存后不生效，被后端按推荐开始 ±2h 重算覆盖；编辑已有操作时同样字段却能正常保存，行为不一致。

根因：ProcessTemplateV3/QuickCreateOperationModal.tsx:859 新建时无条件传 absoluteStartHour；后端 backend/src/controllers/stageOperationController.ts:551-563 见此字段即走 buildRelativeScheduleFields 重算窗、忽略请求里的 window_*。

建议：手动窗模式不传 absoluteStartHour；或后端收到显式 window_* 时优先采用、不重算。

已修复（2026-06-10，后端侧）：把 buildRelativeScheduleFields 拆为 buildPositionFields（落点定位）与 buildDerivedWindowFields（±2h 兜底窗）；请求带显式 window_* 时原样落库不再重算，absoluteStartHour 继续承担画布定位，自动模式行为不变。新增 stageOperationFromCanvas.test.ts 3 用例。边角说明：自动模式 duration=1 时窗结束改按前端预览 +1 落库（修正了预览与存库不一致）。

P1bug已修复A3 · 起始天填 0 的阶段被存成第 1 天

现象：弹窗文案明确写「首日 = 0」且默认填 0，但创建后该阶段实际从第 1 天开始，首阶段及其所有操作整体右移一天。

根因：backend/src/controllers/processStageController.ts:85,97 用 start_day || 1，0 被当假值替换为 1。from-canvas 路径用 ?? 0 是对的，只有 createStage 错。

建议：改为 start_day ?? 1 或显式判空。

已修复（2026-06-10）：两处 start_day || 1 均改为 ?? 1，0 原样保存；同文件其余 || 写法排查后无同类问题（max_order 处属合理防御）。

P1bug已修复A4 · Ctrl/⌘+滚轮缩放不以光标为锚点

现象：按住 Ctrl/⌘ 滚轮缩放时，缩放中心始终在时间轴最左端而非鼠标位置，放大后目标工序跑出视野。

根因：wxb-ui/GanttChart/GanttCanvas.tsx:398 已传 anchorX，但 useGanttStore.ts:98-104 的 ZOOM reducer 根本不读它，只等比缩放 scrollX。

建议：reducer 按锚点世界坐标在缩放后回调 scrollX，保持锚点像素位置不变。

已修复（2026-06-10）：ZOOM reducer 实现锚点保持——newScrollX = clamp(worldX × ratio − anchorX)；anchorX 缺省（工具栏 ± 按钮）回退原等比行为。新增 useGanttStore.test.ts 覆盖锚点/等比/边界 clamp 共 9 用例。

P1perf已修复A5 · 多选拖拽逐条 await 保存且每条都全量刷新

现象：框选多条工序一次拖动，松手后明显卡顿（条数越多越久）；中途失败则前面已写、后面未写，处于半更新不一致状态。

根因：wxb-ui/GanttChart/useGanttDrag.ts:320-327 for-await 串行，且 ProcessTemplateV3/useV3EditorActions.ts:194-198 每条更新后都全量 refreshData()。

建议：批量接口或并发提交后只刷新一次；失败时整体回滚或明确提示部分失败。

已修复（2026-06-10）：wxb-ui 新增可选批量回调 onTasksDragEnd(updates[])（types→GanttChart→GanttCanvas→useGanttDrag 透传），多选拖拽单次批量并发落库 + 只刷新一次 + 部分失败提示；未传该 prop 的消费方（如批次甘特）回退原逐条逻辑，零影响。遗留：Ctrl+Z 对组拖拽仍走逐条还原（独立低优路径）。

P1perf已修复A6 · useGanttData 串行逐阶段请求，N 个阶段发 N+2 次 HTTP

现象：打开模版时加载时间随阶段数线性增长（5 个阶段约 7 次串行请求），白屏/转圈偏长。

根因：ProcessTemplateGantt/hooks/useGanttData.ts:78-81 for 循环内 await。

建议：改 Promise.all 并行拉取。

已修复（2026-06-10）：改为 Promise.all 并行拉取，stage→operations 映射与顺序不变，错误语义保持整体失败，对外返回结构未动。

P1bug已修复B1 · 画布拖长操作条后再开编辑弹窗保存，缩放被静默吞回

现象：把操作条从 4 小时拉长到 8 小时，再双击打开「编辑操作」点保存，结束时间被悄悄改回 4 小时。

根因：BatchGanttV4/index.tsx:453-458 persistTaskTime 只 PUT start/end 不更新 planned_duration；EditOperationModal.tsx:54-65,183 用旧 duration 初始化并以 end = start + duration 重算覆盖，duration 字段又只读不可编辑。

建议：拖拽/缩放时同步写 planned_duration（或后端按 end-start 重算）；弹窗 duration 可编辑或按真实跨度推导。

已修复（2026-06-11，按决策禁用 resize）：工序时长由域模型钉死，批次甘特不再允许拖拽改时长——index.tsx 移除 onTaskResizeEnd、adapter 移除 resizable 标记；wxb-ui 的 resize 命中区/光标改为门控在「传了 onTaskResizeEnd」上（useGanttHitTest 新增 resizeEnabled 参数，默认 true，模版甘特行为不变）。工序条平移（时长不变）保留，planned_duration 与 start/end 不再可能脱节。

P1bug已修复B2 · 分组批量拖拽中途失败留下「半截已落库」的移动

现象：拖动带多条操作的批次/阶段分组，中间某条保存失败时画布整组视觉回弹，但前面几条已写库——界面说没改，库里移了一半。

根因：BatchGanttV4/index.tsx:482-498 for 循环逐条 await PUT，失败 return false 不回滚；成功条目触发的 requestReloadSoon 还会把半截结果拉回画布。

建议：后端原子批量接口；或失败时对已成功操作反向补偿再回弹。

已修复（2026-06-11，原子批量接口）：后端新增 PUT /api/v5/gantt/operations/batch-time——复用单条端点抽出的校验 helper（含硬时间窗），先全量预校验再单事务执行，任一失败 rollback + 400 带 operationId，全有或全无；前端组拖拽与多选拖拽（onTasksDragEnd）均改为单次调用该端点，失败 wxbToast 报因 + 画布回弹、库里零写入。新增 batchGanttOperationsBatch.test.ts 4 用例（成功提交/中途失败全回滚/窗越界不开事务/空数组）。

P1bugB3 · 共享组改成员后甘特连线不刷新

四、被否决的 13 条误报（核查质量记录）

每条 finding 均由独立核查 agent 反驳式复核，以下为被否决的代表性误报及理由：

  • 「级联拖拽撤销链全部失效」——批次侧 index.tsx:635 实际传了 onGroupDragEnd，撤销路径是通的（模版侧的缺口已由 A1 覆盖）。

  • 「人员峰值热力图漏最后一天」——数学上不成立（endOf('day') 的小数小时差覆盖了边界）。

  • 「共享组三数据源口径不一致」——连线与徽标实际只消费 shareService 一个来源。
